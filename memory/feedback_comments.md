Use comments sparingly in code. Only comment complex code.

**Why:** User prefers clean, self-explanatory code with minimal comments.

**How to apply:** When writing or modifying Go code, only add comments for non-obvious logic, algorithms, or workarounds. Avoid commenting simple functions, obvious variable declarations, or standard patterns.
